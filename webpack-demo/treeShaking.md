1.treeShaking会在生产模式下打开

它是一组功能搭配使用后实现的效果，这组功能在生产模式下会自动启用

注意：Tree-shaking使用的前提是ES Modules 必须是这种方式来组织的模块化
使用babel-loader转换js时，可能会把ES Modules处理为common.js的形式  导致Tree-shaking无法正常使用
最新的babel-loader最新版已经处理了这个问题，所以要根据babel-loader版本去做不同的处理

sideEffects
TIPS 也叫模块的副作用，指的是模块执行除了导出成员，是否还做了其它事情
场景：在一个文件对一个文件夹下做了一个共同暴露，然而实际使用的时候只用了其中一个模块，因为是用一个文件对外导出了，会导致所有文件
     都会导出，这样会产生很多冗余代码。
但是很多基于原型的操作，自定义一些函数，会面临被移除，这样运行就会报错，可以在配置中配置哪些文件不被sideEffects

Code Splitting
通过把项目中的资源模块按照我们设计的规则打包到不同的bundle中，降低应用的启动成本，提高响应速度
分包方式：配置多个打包入口，输出多个打包结果
        结合ES Modules的动态导入特性，按需加载模块
多个入口打包存在一个问题，不同入口一定会存在打包相同模块，这个时候就需要chunks。提取公共模块的打包
Code Splitting更常见的实现方式还是结合ES Module的动态导入特性 从而实现按需加载
